เอกสารนี้รวบรวมการออกแบบกติกาเกม (Game Design) และโครงสร้างระบบ (System Architecture) สำหรับบอร์ดเกมทำมือธีม Pokémon แนว Multiplayer

## 🎲 PART 1: Game Design (กติกาและระบบเกม)

เป้าหมายหลัก: **"ผู้เล่นที่มีเงินมากที่สุดตอนจบเกมคือผู้ชนะ"** 

### 1. ระบบอาชีพ (16 Classes System)
ผู้เล่นจะได้สุ่มหรือเลือกอาชีพตอนเริ่มเกม แต่ละอาชีพจะมีจุดเด่นและวิธีหาเงินที่ต่างกัน

### 2. ลำดับการเล่นใน 1 เทิร์น (Turn Phases)
1. **Phase 1: Draw Phase (จั่วการ์ด):** จั่วการ์ดแบบสุ่ม 1 ใบ (สูงสุดถือในมือได้ 3 หรือ 5 ใบ) 
2. **Phase 2: Item Phase (ใช้ไอเทม):** สามารถลงการ์ดไอเทมกี่ใบก็ได้ เพื่อบัฟตัวเอง เติมโปเกบอล หรือสกัดขาคู่แข่ง
3. **Phase 3: Class Phase (ใช้สกิล):** กดใช้สกิล Active หรือยิง Ultimate Skill ถ้าคูลดาวน์พร้อม
4. **Phase 4: Action Phase (ทอยเต๋า):** ทอยเต๋า 2 ลูกเพื่อเดินตามช่อง เมื่อหยุดที่ช่องไหน ให้รับผลของช่องนั้นทันที 
   - *ช่องป่า (Wild): ทอยเต๋าจับโปเกมอน*
   - *ช่องเมือง (City): ขายเป็นเงินหรือซื้อของ*
   - *ช่องยิม (Gym): ทอยเต๋าสู้เอาเงินรางวัลเหรียญตราชนะ*
   - *ช่องอีเวนต์ (Event): สุ่มจั่วการ์ดดวง*

### 3. การ์ดไอเทมยอดฮิต (10 ชนิด)
1. **Bicycle (จักรยาน):** ปั่นฉิว! ทอยเต๋าเดินเพิ่ม 1 ลูกในเทิร์นนี้
2. **Poke Ball:** ลูกบอลพื้นฐาน ใช้โยนจับโปเกมอนทั่วไป (ใช้ 1 ลูก/การจับ)
3. **Ultra Ball:** เพิ่มเรทการจับให้ง่ายขึ้น ถือว่าทอยเต๋า +2 แต้ม
4. **Nugget (ก้อนทองคำ):** ไอเทมขยะทองคำ เก็บไว้เอาไปโยนขายที่ร้านค้าในราคา 3,000 ฿ 
5. **Escape Rope (เชือกหลบหนี):** วาร์ปตัวเองกลับไปที่ "เมือง" ที่ใกล้ที่สุดทันที
6. **Full Heal (ยารักษา):** ล้างสถานะผิดปกติทั้งหมด (หลับ, พิษ, ชา)
7. **Max Repel (สเปรย์ไล่):** กางบาเรีย ไม่ให้คนอื่นใช้สกิลใส่ หรือหนีอีเวนต์ร้ายๆ ได้ 2 เทิร์น
8. **Team Rocket Uniform (ชุดแก๊งร็อคเก็ต):** เนียนเป็นโจรกระชากการ์ด 1 ใบจากมือเพื่อน 
9. **Snorlax Flute (ขลุ่ยปลุกคาบิกอน):** เสกคาบิกอนไปขวางทางบล็อคช่องเดิน 1 เส้นทาง
10. **EXP Share (เครื่องแบ่งประสบการณ์):** แอบดูดเงินเพื่อน! หากเล็งใครไว้ พอมันได้เงิน เราจะได้ด้วย 20%

### 4. ระบบการจับโปเกมอน & ขายโปเกมอน
* **การจับ (Capture System):** เมื่อตกช่อง **"ป่า"** หน้าจอจะโชว์โปเกมอนป่า (แบ่งเกรด Common, Rare, Epic) 
  - ต้องจ่าย โปเกบอล 1 ลูก เพื่อเข้าสู่โหมดจับ
  - **ทอยเต๋า 1 ลูก**: Common ต้องได้แต้ม `3+`, Rare ต้องได้ `4+`, Epic ต้องได้ `6`
  - หากโยนบอลพลาด โปเกมอนจะหนีไป (เสียบอลฟรี) หากได้บอลแพง (Ultra Ball) จะลดแต้มเต๋าที่ต้องการลง
* **การขาย (Trading/Selling):** นำโปเกมอนที่จับได้ไปตกช่อง **"เมือง/ร้านค้ารับซื้อ"** 
  - Common ขายได้ `1,000 ฿` 
  - Rare ขายได้ `3,000 ฿` 
  - Epic หรือร่างพัฒนา ขายได้ `6,000 ฿` 
  - *ข้อควรระวัง:* บางเทิร์นตลาดอาจมีการ **"ปั่นราคา"** (เศรษฐกิจแกว่ง ธาตุไฟราคาขึ้น ธาตุน้ำราคาตก)

## 💻 PART 2: System Architecture (โครงสร้างเบื้องหลัง)

### 1. Data Structure & Schema
ในการทำ WebSocket ควรส่งข้อมูลสเตทยกแผงเป็น JSON:

```json
// GameState Object (สเตทของทั้งกระดาน)
{
  "roomId": "pokemon_room_99",
  "turnCount": 12,
  "currentPlayerIndex": 0,    // ระบุว่าเทิร์นของใคร
  "currentPhase": "DRAW",     // DRAW, ITEM, CLASS, ACTION
  "players": [                // ข้อมูลผู้เล่น 4 คน
    {
      "playerId": "p1",
      "name": "Satoshi",
      "classId": "class_rookie",
      "position": 5,          // อยู่ช่องที่ 5 บนกระดาน
      "money": 1500,
      "cards": ["item_bicycle", "item_pokeball"], // การ์ดบนมือ
      "pokemons": ["pkmn_caterpie"],              // มอนสเตอร์ในกระเป๋า (รอขาย)
      "statusModifiers": [],                      // สถานะเช่น ['POISONED', 'SLEEP']
      "ultimateCooldown": 0
    }
    // ... players 2, 3, 4
  ],
  "boardEvents": [            // สิ่งที่วางอยู่บนบอร์ด (เช่น กับดักยาสลบของนักจับแมลง)
    { "tileId": 8, "type": "TRAP_POISON", "owner": "p2" }
  ]
}
```

```json
// Card / Item System (ตัวอย่างข้อมูลการ์ด 1 ใบ)
{
  "cardId": "item_escape_rope",
  "name": "Escape Rope",
  "type": "TELEPORT",
  "targetType": "SELF",     // SELF, OPPONENT, TILE, NONE
  "effect": {
    "action": "MOVE_TO_NEAREST",
    "value": "CITY"
  }
}
```

### 2. Core Game Loop (Server Backend Flow)
เซิร์ฟเวอร์จะทำหน้าที่เป็น "กรรมการ" เช็คสเตทแบบ State Machine วน Loop 4 Phase:
1. **DRAW PHASE**: 
   - Backend เช็คว่าเข้าเทิร์น P1
   - Backend สุ่มหยิบการ์ดจาก Deck ใส่ `players[0].cards` เข้าไป
   - ส่ง WebSocket `update_game_state` ไปให้ทุกคน
   - ดัน GameState สู่เฟส `ITEM_PHASE` ทันที
2. **ITEM PHASE**:
   - Backend รอรับ Event `use_item` จาก Client หาก P1 กดใช้ของ
   - คำนวณผลลัพธ์ (เช่น ให้เงิน บวกลบสเตตัส) 
   - เมื่อ P1 กดต่อ จะส่ง Event `end_phase` มา -> เปลี่ยนเป็น `CLASS_PHASE`
3. **CLASS PHASE**:
   - รอรับ Event `use_skill` หากต้องการใช้สกิล
   - เช็ค Cooldown ว่าพร้อมไหม ถ้าครบก็แสดงผลแอนิเมชันให้ทุกคนเห็น
   - P1 กด `end_phase` -> เปลี่ยนจรวดไป `ACTION_PHASE`
4. **ACTION PHASE**:
   - รอรับ Event `roll_dice` สุ่มเลขเต๋าที่ backend (กันโดนแฮคโกงเต๋าฝั่ง client)
   - ขยับตำแหน่ง P1 ชนกับใครไหม? ร่วงลงช่องอีเวนต์ไหน?
   - ถ้าตกช่องจับโปเกมอน -> เข้าสู่ Sub-phase `ENCOUNTER` รอรับ event การโยนลูกบอล
   - สุดท้ายเมื่อจบทุกอย่าง ตัดจบเทิร์น ย้าย `currentPlayerIndex` ไปหา P2

### 3. API & WebSocket Events
ใช้ Socket.io สื่อสารแบบ Real-time:

**🟢 Client Emits (ส่งจาก Frontend ไป Server):**
* `join_room` (roomId, playerName)
* `select_class` (classId)
* `play_card` (cardId, targetPlayerId)
* `use_skill` (skillId, target)
* `roll_dice` ()
* `throw_pokeball` (pokeballType)
* `sell_pokemon` (pokemonId)
* `end_phase` ()

**🔵 Server Broadcasts (ส่งจาก Server ไปอัปเดตพาเนลผู้เล่น):**
* `game_started` (GameState เริ่มต้น)
* `update_game_state` (ส่ง GameState ทับสเตทเดิมเสมอ ให้ Frontend Render ตามข้อมูลนี้)
* `dice_rolled` (player, result, targetTile) - สำหรับเล่นแอนิเมชันลูกเต๋ากลิ้ง
* `action_feedback` (type, message) - แจ้งเตือนเช่น "Satoshi ถูกพิษมอนสเตอร์ตายคาที่!"
* `encounter_started` (pokemonData, catchRate) - เปิดป็อปอัปสู้มอน

### 4. Tech Stack Recommendation
เพื่อให้เกมลื่นไหลและรองรับ Multiplayer ได้เสถียร:

* **Frontend:** `React (Next.js)` + `TailwindCSS`
  - *เหตุผล:* เขียน UI จัดการการ์ด หน้าต่างแชท หน้าต่างข้อมูลผู้เล่นได้เร็วมาก
* **Game Board / Canvas Engine:** `Pixi.js` หรือ `Phaser.io`
  - *เหตุผล:* เอาไว้เรนเดอร์กระดานบอร์ดเกม ทอยเต๋า เดินหมากเป็นช่องๆ (React เฉยๆ อาจจะทำอนิเมชั่นการเดินเหนื่อยไป ใช้ WebGL Engine ช่วยจะสวยและลื่นระดับ 60 FPS)
* **State Management (Client):** `Zustand` สำรอง state เผื่อ socket กระตุก
* **Backend:** `Node.js` + `Express` + `Socket.io`
  - *เหตุผล:* การใช้ JavaScript ทั้งหน้าบ้านและหลังบ้าน ทำให้ Share logic ส่วนของการคำนวณ (เช่น ลูกเต๋า สกิล) ด้วยโค้ดชุดเดียวกันได้ Socket.io มีระบบ Room ที่จัดการปาร์ตี้ 4 คนได้เพอร์เฟค
* **Database (State Sync):** `Redis` 
  - *เหตุผล:* เก็บ GameState บน Redis In-Memory เพราะบอร์ดเกมมีการอัปเดตสถานะ ทุกๆ 1 วินาที! การเซฟลง DB ปกติจะทำให้ช้าเกินไป (ดึงจาก Redis จะเร็วระดับ Milliseconds) ส่วน `MongoDB` เก็บแค่ประวัติชนะแพ้และบัญชีผู้เล่นพอครับ
